# Cognitive Orchestration Test (COT)

An open-source, local-first situational behavioral matrix designed to map how different minds naturally organize data, manage entropy, and interact with structured logic.

## 🧠 The Framework

Instead of relying on traditional human psychology metrics, this test maps operational behaviors onto three primary architectural axes by observing how users handle chaos in everyday situational simulations.

### 1. The Operational Axis
* **System Scaffolder (Architect):** High affinity for structured formatting, explicitly defined constraints, and clear data isolation.
* **Granular Iterationist (Co-Pilot):** High affinity for low-latency feedback loops, incremental modifications, and agile course-correction.

### 2. The Adversarial Axis
* **Edge Auditor (Red-Teamer):** Natural instinct to probe boundaries, locate structural loopholes, and test alignment limits through lateral reframing.
* **Compliant Transactionalist:** High efficiency within standard parameters; operationalizes rules exactly as written without testing stress points.

### 3. The Relational Axis
* **Mechanical Functionalist:** Strips out communicative grease; prioritizes high information density and direct imperative execution.
* **Anthropomorphic Bridge:** Utilizes high social grease and conversational frameworks to mirror internal reasoning out loud.

---

## 🛠️ Repository Mapping

* `index.html` / `style.css` — Core interface layer (vanilla, zero dependencies).
* `src/scenarios.json` — Immutable dataset containing the situational modules and vector weight balances.
* `src/evaluator.js` — The logic processing engine that aggregates choice points and calculates the dominant profile archetype.
* `profiles/` — Deep-dive informational breakdowns of each resulting cognitive alignment vector.

## 🚀 Execution
This project runs entirely client-side using native web technology. No external APIs, databases, or complex build pipelines are required.
